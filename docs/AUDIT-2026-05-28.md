# claude-sync security & correctness audit ‚Äî 2026-05-28

Multi-agent audit across three rounds (12 specialists). Findings deduplicated and
grouped by area, severity-sorted within each group. Design choices (convenience
mode existing as a feature, framework choices, module layout) are accepted; this
is a defect list, not a redesign.

Severity legend: **C** critical, **H** high, **M** medium, **L** low.

---

## 1. Auth & crypto

- **[C] Scrypt parameter downgrade** ‚Äî `server/src/auth/password.ts:26-34`. `verifyPassword` trusts `N/r/p` from the stored hash with only `Number.isFinite` checks. An attacker with DB write access can store `N=1` (trivial brute force) or `N=2^20, r=64, p=64` (per-login DoS, 256 MiB / multi-second CPU). Cap before calling scrypt: `if (n < 1<<14 || n > 1<<17 || r < 8 || r > 8 || p < 1 || p > 1) return false`.
- **[C] Convenience mode defeats E2E guarantee** ‚Äî `desktop/src/main/index.ts:146-179`. Vault key = argon2id(login password). Server sees the password in plaintext during `/auth/login` and stores the kdf salt, so the operator can derive the vault key for any user with convenience mode on. Default it off and remove the "E2E encrypted" claim while it's enabled.
- **[C] GitHub OAuth email is trusted as verified** ‚Äî `server/src/auth/github.ts:99,147-148`. `/user` returns the primary email regardless of verification state. An attacker can register an account under any unverified email, then refuse to release the GitHub email ‚Äî account takeover before victim's first login. Require verified-email scope and check `verified` on each email from `/user/emails`.
- **[H] Scrypt blocks all 4 libuv crypto threads** ‚Äî `server/src/auth/password.ts` + `email.ts`. Default Node thread pool size 4; each scrypt call is ~134 MiB / ~500 ms. 4 concurrent auth requests saturate the pool and block all other crypto work (including HTTPS handshakes if not terminated upstream). Move to `worker_threads` with a semaphore, or set `UV_THREADPOOL_SIZE=8` as a stopgap.
- **[H] Signup hashes before duplicate-email check** ‚Äî `server/src/auth/email.ts:28`. Every signup pays the full scrypt cost even on a duplicate email. Reorder: SELECT ‚Üí hash ‚Üí INSERT.
- **[H] Account enumeration via `/auth/signup`** ‚Äî `server/src/auth/email.ts:38-40`. HTTP 409 "email already registered" is a direct existence oracle. Login is correctly uniform; signup is not.
- **[H] Convenience mode passes login password to sync process** ‚Äî `desktop/src/main/index.ts:88-120, 220`. `passphraseRef.value` stores the cleartext password for the whole session and re-derives the key every sync cycle, even though the derived key is already in memory. Heap dump / debugger attach yields the password, which unlocks the vault on every device. Thread `state.vaultKey` through `cliPull`/`cliPush` and drop `passphraseRef`.
- **[H] Session tokens stored unhashed in DB** ‚Äî `server/src/auth/session.ts:17`. DB read = live session tokens. Hash before storage (SHA-256), hash on lookup.
- **[H] Vault key cannot be rotated** ‚Äî `server/src/routes/vault.ts:50-57`. Once `count > 0`, key is permanent. Compromised passphrase = account deletion (and no delete endpoint exists either).
- **[H] CSRF mitigation missing on `/auth/*`** ‚Äî `server/src/lib/security-headers.ts:11-16`. `X-Requested-With` check only applies to `/api/*`. `/auth/logout` and `/auth/sessions/revoke-all` are CSRF-able (DoS); `/auth/login` is session-fixation-able. SameSite=strict mitigates most cases but not all (redirect chains).
- **[H] OAuth callback redirects to env-controlled URL with no same-origin check** ‚Äî `server/src/auth/github.ts:105`. A misconfigured `AUTH_URL` (e.g., dev value in prod) sends fresh session cookies to wrong host.
- **[M] OAuth state cookie split is fragile** ‚Äî `server/src/auth/github.ts:75-80`. `split('.')` on a multi-dot value truncates the signature; `Buffer.from(undefined)` would throw rather than return false (the `!cookieSig` guard saves it today). Use `lastIndexOf('.')` and length-validate both halves.
- **[M] GitHub `email: null` not handled consistently** ‚Äî `server/src/auth/github.ts:110-122`. Stored as NULL; client types it as `string`; `.toLowerCase()` on a null email crashes the client.
- **[M] `clearSessionCookie` omits attributes** ‚Äî `server/src/auth/session.ts:71`. RFC 6265 says clearing must echo `Secure`/`HttpOnly`/`SameSite`/`Path`. Some browser+proxy combos may not delete the cookie on logout.
- **[M] No transaction around signup user+session creation** ‚Äî `server/src/auth/email.ts:31-36`, `server/src/auth/github.ts:99-102`. If the process dies between `INSERT users` and `ensureUserSeqRow`, the user exists but can never upload (`user_seq missing` 500 forever). Wrap in `db.transaction`.
- **[L] Session middleware bumps `last_seen_at` on every request** ‚Äî `server/src/auth/session.ts:44-46`. Dozens of writes/sec/user. Debounce with `WHERE last_seen_at < now() - interval '1 minute'`.

## 2. Crypto correctness

- **[H] Path traversal on pull** ‚Äî `cli/src/commands/pull.ts:34`. `c.path` is server-supplied; `join(syncRoot, "../../../.bashrc")` resolves outside the root. Re-canonicalize on the client with the existing `canonicalizePath` before any filesystem write.
- **[H] Empty `path` passes the `!c.path` guard** ‚Äî `cli/src/commands/pull.ts:34`. A path of `""` is falsy and skipped; but if it ever reached the join, `join(syncRoot, "")` returns `syncRoot` itself, overwriting the directory. Belt-and-suspenders: type `path` as `NonEmptyString` after validation.
- **[H] Missing header validation in `getLatest` / `restore`** ‚Äî `cli/src/api.ts:121-137`. `nonceB64`, `keyId`, `versionId` default to `""` if the server omits the header. Empty nonce decodes to a 0-byte buffer; libsodium throws an opaque error inside the crypto path with no context. Validate each header after read; throw `ApiError` on missing.
- **[H] AAD positional fields are unbranded `string`** ‚Äî all IDs (userId, fileId, versionId, keyId, deviceId) are plain `string`. Transposing any two compiles cleanly but produces wrong AAD ‚Üí silent decryption failure or, in oracle scenarios, plaintext bound to the wrong identity. Use branded types.
- **[H] `userId` is `string | undefined` past the guard** ‚Äî `cli/src/commands/pull.ts:63`, `cli/src/commands/push.ts:76`. TS doesn't narrow object property access reliably. Move to a `ReadyConfig` type with non-optional fields.
- **[M] Salt truncation silently lossy** ‚Äî `cli/src/crypto.ts:32-33`, `web/src/crypto.ts:23`. Server allows up to 32-byte salts but clients `slice(0, 16)`. Today both clients truncate identically so keys agree, but a future client that uses the full salt would derive a different key. Generate only 16-byte salts and reject longer-stored ones.
- **[M] Tombstone min-ciphertext check accepts unbounded payloads** ‚Äî `server/src/routes/files.ts:96-99`. `>= 16` is a floor; there's no ceiling, and quota is skipped for `deleted=true` (`files.ts:163`). A malicious client can store huge ciphertext under DELETE bodies. Cap tombstone bodies at `POLY1305_TAG + 16` or charge them to quota.
- **[M] Comment self-contradiction on `allocateSeq`** ‚Äî `server/src/routes/files.ts:158-160`. Comment says rolled-back tx "still consumes a seq" then claims ROLLBACK restores. Rollback actually restores correctly (no gap), but the comment will mislead someone into "fixing" it with a Postgres sequence, which won't roll back.
- **[L] No end-to-end AEAD-binding test** ‚Äî `aeadDecrypt` substitution-attack invariant has only the `buildAad` unit test; cross-file substitution is the central crypto invariant and has no integration test.

## 3. Server routes & validation

- **[C] GitHub OAuth response cast without runtime validation** ‚Äî `server/src/auth/github.ts:147`. `as { id: number; email: string | null }` on `res.json()`. A 200 with `{ }` produces `INSERT ... NULL` into `github_id NOT NULL` ‚Üí unhandled 500. Validate with Zod.
- **[H] `/api/files/:fileId/versions` unbounded** ‚Äî `server/src/routes/files.ts:249-256`. No LIMIT; long-running users yield arbitrarily large arrays. Add cursor pagination.
- **[H] `GET /api/files` unbounded** ‚Äî `server/src/routes/files.ts:70-81`. Same; users with thousands of files OOM the response. Page with `LIMIT` + cursor.
- **[H] Activity endpoint always re-scans from `seq=0`** ‚Äî `desktop/src/main/index.ts:416`, `web/src/app.ts:260`. Both clients send `since=0&limit=500` every activity refresh, forcing a full-history scan to return 50 rows. Pass `since = max(0, latestSeq - 500)` or add `/api/activity?limit=50`.
- **[H] Quota check is full-partition scan inside the row-locked tx** ‚Äî `server/src/routes/files.ts:163-172`. `SUM(size_bytes) WHERE NOT deleted` with no covering index. Add partial index `(user_id) WHERE NOT deleted` or maintain `users.storage_bytes` as a running total.
- **[H] `Cache-Control: no-store` missing on auth responses** ‚Äî `server/src/lib/security-headers.ts`. A misconfigured intermediate proxy could cache a `Set-Cookie` response and serve the session to the next requester.
- **[H] `req.body` not in pino redact paths** ‚Äî `server/src/lib/redact.ts`. Headers and ciphertext are redacted; bodies aren't. Anyone adding `req.log.info(req.body)` for debugging will dump passwords to Railway logs. Add `req.body.password`, `req.body.email`, and `*.password`.
- **[M] `@fastify/static` mounted at `/` can shadow API 404s** ‚Äî `server/src/lib/web.ts:21`. SPA fallback returns HTML for any path. CLI clients get HTML where they expect JSON; a malicious bundle path could shadow an API route. Restrict prefix or use a `setNotFoundHandler` that defers `/api/*`/`/auth/*` to a JSON 404.
- **[M] Server query string logged including file paths** ‚Äî `server/src/lib/redact.ts`. `req.url` with `?path=skills/secret/...` lands in logs, defeating the "server-blind on paths is logged separately anyway" assumption.
- **[M] Rate-limit `keyGenerator` runs before `preHandler`** ‚Äî `server/src/lib/rate-limit.ts:22-25`. `req.user` is always `undefined`, so all `/api/*` rate limiting is per-IP, not per-user. Users behind a shared NAT share the bucket; the env var name `RATE_LIMIT_API_PER_USER` is misleading.
- **[M] `/api/sync` is unauthenticated to rate-limit but lists every path** ‚Äî `server/src/routes/sync.ts:22-27`. Compromised session can enumerate the entire path tree quickly. Apply the per-IP/user limit and consider path inclusion in AEAD if you want true server-blind paths.
- **[M] Devices delete + session revoke not atomic** ‚Äî `server/src/routes/devices.ts:57-61`. Two separate statements; concurrent request sees a deleted device with a live session.
- **[M] `upsertUser` is select-then-insert-then-select** ‚Äî `server/src/auth/github.ts:110-122`. Three round-trips on the common path. Use `INSERT ‚Ä¶ ON CONFLICT (github_id) DO UPDATE ‚Ä¶ RETURNING id`.
- **[L] Healthz returns HTTP 200 when DB is down** ‚Äî `server/src/index.ts:57-64`. Body says `db: 'down'` but status is 200; load balancers won't drain. `reply.code(503)`.
- **[L] No `pool.on('error')` handler** ‚Äî `server/src/db/client.ts`. Idle-client error on `pg.Pool` is an EventEmitter error ‚Üí process crash. Add a no-op-with-log listener.
- **[L] Pool defaults ‚Äî no `idleTimeoutMillis`, no `connectionTimeoutMillis`** ‚Äî `server/src/db/client.ts:10`. Hung queries lock the pool forever. Set both.
- **[L] No statement_timeout / idle_in_transaction_session_timeout** ‚Äî `server/src/db/client.ts:20-43`. A slow tx with `FOR UPDATE` can block every concurrent upload from that user. `SET LOCAL` inside `BEGIN`.
- **[L] `limit=NaN` reaches Postgres as `LIMIT NaN`** ‚Äî `server/src/routes/sync.ts:6`. `Math.min/max(NaN, ...)` is NaN. Validate with Zod coerce.
- **[L] Server fingerprint header** ‚Äî `Server: Fastify/5.x.x` not stripped. Strip in the `onSend` hook.

## 4. Database schema

- **[C] `file_versions.user_id` has no FOREIGN KEY** ‚Äî `server/src/db/migrations/0001_init.sql:65`. Orphan rows possible if any path bypasses the `files` cascade.
- **[H] Missing index `(user_id) WHERE NOT deleted` on `file_versions`** ‚Äî used for quota SUM on every upload, no index.
- **[H] Missing index on `devices(user_id)`** ‚Äî every `GET /api/me` does a seq scan.
- **[H] Migrations run on every server start with no tracking table** ‚Äî `server/src/db/migrate.ts:9-14`. Idempotency relies on `IF NOT EXISTS` in every DDL. A future migration with `ALTER` or data backfill will re-run and break. Two replicas race with no advisory lock. Add `schema_migrations` + per-file `BEGIN/COMMIT` + `pg_advisory_xact_lock`.
- **[M] `users.github_id` UNIQUE with multiple NULLs** ‚Äî `server/src/db/migrations/0002_email_password.sql`. Postgres treats each NULL as distinct, so the uniqueness contract weakens silently after dropping NOT NULL. Document or use `EXCLUDE` constraint.
- **[L] `files.path` has no `CHECK (path IS NULL OR length(path) > 0)`** ‚Äî empty string would satisfy the partial unique index and pass DB checks.
- **[L] `file_versions.key_id` has no FK to `vault_key_metadata`** ‚Äî orphan possible if metadata is ever deleted.

## 5. Sync correctness (client)

- **[C] Pull cursor advances in memory before files are durably written** ‚Äî `cli/src/commands/pull.ts:24-95`. `cursor` reassigned per item; `saveConfig` only at end of loop. Crash mid-batch ‚Üí cursor in memory not flushed, files on disk; on resume, replays and may trigger conflict logic. Persist `cursor` + manifest after each successfully applied change.
- **[C] Push has no idempotency on retry** ‚Äî `cli/src/commands/push.ts:73-87`, `cli/src/api.ts:105-119`, `server/src/routes/files.ts:145-194`. Network flap after a successful PUT ‚Üí client generates a new `versionId` next push ‚Üí orphan version that still counts against quota. Derive `versionId` deterministically from `(fileId, plaintextHash)` or split the 409 from 404.
- **[C] Watch loop catches everything and busy-loops every 10s** ‚Äî `cli/src/commands/watch.ts:9-20`. Single corrupt remote version blocks all push direction indefinitely; auth failures hammer the rate limit until banned. Per-op try/catch + exponential backoff + skip-with-log on undecryptable versions.
- **[H] Pull `deleted++` and `delete manifest[c.path]` run even when `rename`/`unlink` threw** ‚Äî `cli/src/commands/pull.ts:52-56`. Conflict-copy rename failed but manifest is already updated ‚Üí local file is orphaned. Move increments inside the `try`.
- **[H] Tombstone with `path=null` skipped on pull** ‚Äî server clears `files.path` on tombstone (`files.ts:140`); sync feed returns `path=null` for the tombstone change; client's `if (!c.path) continue` (line 34) skips it ‚Üí local file never deleted. Carry last-known path in the tombstone event, or look up by `file_id` in the manifest.
- **[H] `convenienceMode` toggle silently dropped** ‚Äî `desktop/src/main/index.ts:349-356`. Handler declares it in the type but never writes it. Settings UI checkbox is non-functional. Add the assignment.
- **[H] Conflict-detection on overwrite is untested** ‚Äî `cli/src/commands/pull.ts:66-78`. No test anywhere. A regression silently overwrites local edits.
- **[H] Conflict-detection on delete-vs-edit is untested** ‚Äî `cli/src/commands/pull.ts:37-55`. Same; silently discards local edits.
- **[H] Concurrent PUT from same user is untested** ‚Äî `allocateSeq` row-lock has only sequential coverage. The system's most lock-sensitive invariant.
- **[H] Quota boundary is untested** ‚Äî only the bulk-threshold case is tested; off-by-one in `>` vs `>=` would not be caught.
- **[M] Push partial-failure swallowed per file** ‚Äî `cli/src/commands/push.ts:85-87`. Prints to stderr and continues; caller sees success even when most files failed. `syncOnce` never sets `state.lastError`. Accumulate failures and surface.
- **[M] `walk()` silently skips unreadable directories** ‚Äî `cli/src/commands/push.ts:14-23`. Conflates ENOENT (fine) with EACCES (operator error). Branch on `err.code`.
- **[M] Conflict-detection TOCTOU window** ‚Äî `cli/src/commands/pull.ts:38-79`. `existsSync` ‚Üí `readFile` ‚Üí hash ‚Üí `writeFile`. User save between hash and write is silently overwritten.
- **[M] `makeConflictPath` TOCTOU + sync I/O** ‚Äî `cli/src/commands/pull.ts:100-108`. `existsSync` probe race; also blocks the Electron main thread. Switch to a nanosecond-suffix scheme.
- **[M] Delete-path catch swallows all errors, not just ENOENT** ‚Äî `cli/src/commands/pull.ts:51`.
- **[M] Pull batch dedupes intermediate tombstones via `latestByFile`** ‚Äî `cli/src/commands/pull.ts:29-31`. Within a 200-item page, an upload‚Üídelete‚Üíupload sequence loses the intermediate tombstone. Comment doesn't document this; design-intentional but should be noted.
- **[M] Pull reads each file twice for conflict detection** ‚Äî `cli/src/commands/pull.ts:38-50, 67-73`. 400 sequential readFiles per 200-file batch + redundant hashing. Cache per iteration.
- **[L] Off-by-16 in push size check** ‚Äî `cli/src/commands/push.ts:62`. `1MiB - 32` should be `1MiB - 16`; files in `[1048544, 1048560)` are unnecessarily skipped.
- **[L] `loadConfig` / `loadManifest` no schema validation** ‚Äî `cli/src/config.ts:53,78`. Corrupted/hand-edited JSON propagates as wrong-typed values: `cursor: "abc"`, missing `fileId` ‚Üí unbounded version creation, etc. Add Zod schemas.
- **[L] `config.json` written without `chmod 600`** ‚Äî `cli/src/config.ts:51-64`. Session cookie + KDF salt + (sometimes) DPAPI-encrypted passphrase end up world-readable on multi-user systems.

## 6. Electron / desktop

- **[H] `savePassphrase` silently returns when `safeStorage` unavailable** ‚Äî `desktop/src/main/index.ts:124-129`. User asks to remember, function returns without saving, no error to caller. Throw on `!isEncryptionAvailable()`.
- **[H] No `will-navigate` / `setWindowOpenHandler`** ‚Äî `desktop/src/main/index.ts:68-76`. `sandbox: true` mitigates, but defence-in-depth for security-sensitive app should reject any cross-origin navigation.
- **[H] Auto-updater errors fully discarded** ‚Äî `desktop/src/main/index.ts:501-502`. `.catch(() => {})` on the periodic check. Signature failure indistinguishable from "no update." Log + surface as `state.lastError`.
- **[H] No publisher pinning on `electron-updater`** ‚Äî `desktop/package.json:74-81`. NSIS Authenticode verification skipped without `nsis.publisherName`. Anyone with write access to the GH release ships arbitrary code to all users.
- **[H] `loadRememberedPassphrase` returns `null` on any decrypt error** ‚Äî `desktop/src/main/index.ts:138`. OS keystore reset / wrong-profile / corrupted ciphertext indistinguishable from "no passphrase saved." Auto-unlock silently fails on startup.
- **[H] `auth:login` convenience block is log-and-continue** ‚Äî `desktop/src/main/index.ts:252-254`. Returns `{ user }` as if login succeeded; sync loop never starts; UI shows logged in, no errors. Propagate or set `state.lastError`.
- **[H] All IPC handlers accept untrusted renderer payload without runtime validation** ‚Äî `desktop/src/main/index.ts:205,229,272,281,311,349,381,393,427,437,481,488`. TS types are compile-time only; renderer compromise/XSS sends arbitrary shapes. `settings:set` with `syncRoot: "../../"` is the worst ‚Äî bypasses canonicalize entirely. Zod-parse every handler.
- **[H] `auth:signup` convenience block has no try/catch** (unlike `auth:login`) ‚Äî `desktop/src/main/index.ts:205-226`. Failure surfaces as "object could not be cloned" generic error.
- **[H] `deriveVaultKey` blocks the Electron main thread** ‚Äî `desktop/src/main/index.ts:163,176,292,306,316,535`. 300-800 ms freeze of all IPC + tray + window events on every call. Move to `worker_threads`.
- **[M] `safeStorage`-encrypted passphrase persisted next to session cookie** ‚Äî `desktop/src/main/index.ts:123-138`, `config.json`. Any same-user process can call `safeStorage.decryptString` on the blob (DPAPI scopes to user, not app). Same-machine malware gets both the cookie and the passphrase. Store in OS keychain proper (`keytar`) or don't persist.
- **[M] Logout server call swallowed with `/* ignore */`** ‚Äî `desktop/src/main/index.ts:263`. Server-side session revocation failure invisible; cookie stays valid on server.
- **[M] `setInterval` for auto-update never cleared on shutdown** ‚Äî `desktop/src/main/index.ts:502`. Can fire during quit teardown.
- **[M] `startSyncLoop` uses `setInterval`, drops ticks on long syncs** ‚Äî `desktop/src/main/index.ts:107-109`. Long push pauses the loop entirely with no spacing. Use `setTimeout`-chain.
- **[L] `refreshSettings()` has no try/catch** ‚Äî `desktop/src/renderer/app.js:237-243`. Unhandled rejection swallowed by renderer.
- **[L] `renderChipList` re-reads settings on every chip removal click** ‚Äî `desktop/src/renderer/app.js:253-256`. Rapid clicks race; second read sees stale state, first removal silently lost.

## 7. Web app

- **[H] No CSP / X-Frame-Options / Referrer-Policy** ‚Äî `server/src/lib/security-headers.ts`. Web app handles vault unlock + decryption in the browser; an XSS that exfiltrates `state.vaultKey` is catastrophic and there's no second layer. Strict CSP, `X-Frame-Options: DENY`, `Referrer-Policy: strict-origin-when-cross-origin`.
- **[H] `innerHTML` use is defensively escaped today but fragile** ‚Äî `web/src/app.ts:189,211,268`. Numbers are safe; any future field added without `esc()` is XSS with no CSP backstop. Switch to `textContent` or React-style escaping.
- **[H] `deriveVaultKey` blocks browser main thread** ‚Äî `web/src/crypto.ts:22-26`. 400-1500 ms freeze on desktop; >3 s on mobile. Move to a Web Worker.
- **[H] No tests in `web/`** ‚Äî every web invariant (key cleared on logout, key not in localStorage, decrypt errors handled) is asserted only by code reading.
- **[M] `password` input lacks `autocomplete` attribute** ‚Äî `web/src/index.html:22`. Browsers may autofill old passwords; in convenience mode this is the vault key. Set `autocomplete="current-password"` on login and `new-password` on signup.
- **[M] `boot()` catch shows the auth form on any failure** ‚Äî `web/src/app.ts:42-44`. A 500 from `/api/me` is shown as "signed out." User re-enters credentials, hits the same 500.
- **[M] `renderDiff` appends line-by-line, no `DocumentFragment`** ‚Äî `web/src/app.ts:333-350`. O(n) reflow on large files. Build fragment, then single append.
- **[L] Logout network error fully swallowed (empty `{}`)** ‚Äî `web/src/app.ts:293-296`. No log line either.
- **[L] Boot uses `credentials: 'include'` but comment claims it's the default** ‚Äî `web/src/api.ts:1-3`. Default is `same-origin`. Functionally identical today; misleading if the web app moves off origin.

## Suggested fix order

1. **C-severity in ¬ß1, ¬ß2, ¬ß3, ¬ß5** ‚Äî auth/crypto/cursor/idempotency. These are the load-bearing security and data-integrity invariants.
2. **H-severity in ¬ß1 (account enumeration, OAuth email trust, threadpool exhaustion) and ¬ß3 (cache/log redaction, unbounded queries)** ‚Äî internet-facing exposure.
3. **H-severity in ¬ß5 (manifest drop on rename failure, tombstone null path) and ¬ß6 (IPC validation, auto-updater pinning)** ‚Äî silent data loss / supply chain.
4. **Test coverage gaps from ¬ß5** ‚Äî particularly conflict detection on both paths and concurrent-PUT serialization, then add the AEAD substitution invariant test.
5. **M / L** ‚Äî polish pass after.

## Out of scope (per user)

- Architectural redesign (module layout, framework choices, convenience-mode existing as a feature).
- Renaming/restructuring for taste.
- Switching ORMs (the schema uses raw `pg`, not Drizzle ‚Äî one auditor was misinformed; the schema findings still apply).
